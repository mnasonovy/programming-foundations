# Работа с Git через Git Bash

---

В этом разделе рассмотрим базовую работу с репозиторием через **Git Bash**.

Вы научитесь:

- клонировать репозиторий;
- получать обновления;
- сохранять изменения;
- отправлять изменения в удалённый репозиторий;
- использовать базовые команды Git.

---

## Шаг 1. Клонирование репозитория

### 1.1. Скопируйте ссылку на репозиторий

На странице вашего fork нажмите:

> **Code → HTTPS → скопировать ссылку**

![img.png](../tools%20materials/git%20&%20github/Основные%20команды%20для%20работы%20с%20Git%20через%20консоль(Git%20Bash)/img.png)

### 1.2. Откройте Git Bash

Перейдите в папку, где хотите хранить проект, нажмите правой кнопкой мыши и выберите:

> **Git Bash Here**

![img_1.png](../tools%20materials/git%20&%20github/Основные%20команды%20для%20работы%20с%20Git%20через%20консоль(Git%20Bash)/img_1.png)

---

### 1.3. Выполните команду клонирования

В открывшемся терминале выполните команду:

```
git clone <ссылка на репозиторий>
```

Пример:

    git clone https://github.com/username/programming-foundations.git

![img_2.png](../tools%20materials/git%20&%20github/Основные%20команды%20для%20работы%20с%20Git%20через%20консоль(Git%20Bash)/img_2.png)

После выполнения появится папка с проектом.
![img_3.png](../tools%20materials/git%20&%20github/Основные%20команды%20для%20работы%20с%20Git%20через%20консоль(Git%20Bash)/img_3.png)

---

## Шаг 2. Переход в папку проекта

После клонирования репозитория необходимо перейти в папку проекта,
чтобы выполнять команды Git внутри него.

Выполните команду:

```
cd programming-foundations
```

![img_4.png](../tools%20materials/git%20&%20github/Основные%20команды%20для%20работы%20с%20Git%20через%20консоль(Git%20Bash)/img_4.png)

Название папки соответствует имени репозитория.  
Если ваш репозиторий называется иначе — используйте соответствующее имя папки.

---

## Шаг 3. Проверка статуса репозитория

Чтобы посмотреть текущее состояние репозитория, выполните команду:

```
git status
```
![img_5.png](../tools%20materials/git%20&%20github/Основные%20команды%20для%20работы%20с%20Git%20через%20консоль(Git%20Bash)/img_5.png)
Команда показывает:

- какие файлы были изменены;
- какие файлы добавлены в индекс (staged);
- есть ли незафиксированные изменения;
- есть ли коммиты, которые ещё не отправлены в удалённый репозиторий.

---

## Шаг 4. Получение обновлений из репозитория

Перед началом работы рекомендуется обновить локальную копию репозитория,
чтобы получить последние изменения.

Выполните команду:

```
git pull
```

Эта команда:

- скачивает последние изменения из удалённого репозитория;
- объединяет их с вашей локальной версией проекта.

![img_6.png](../tools%20materials/git%20&%20github/Основные%20команды%20для%20работы%20с%20Git%20через%20консоль(Git%20Bash)/img_6.png)
> Регулярное выполнение `git pull` помогает избежать конфликтов и работать с актуальной версией кода.

---

## Шаг 5. Добавление изменений

После внесения изменений в файлы необходимо добавить их в индекс
(подготовить к сохранению в репозитории).

Выполните команду:

```
git add .
```

Эта команда:

- добавляет все изменённые и новые файлы в индекс;
- подготавливает их к созданию коммита.

> Символ `.` означает, что будут добавлены все изменения в текущей директории и её подкаталогах.

---

## Шаг 6. Создание коммита

После добавления изменений необходимо зафиксировать их в репозитории,
создав коммит.

Выполните команду:

```
git commit -m "Описание изменений"
```

Пример:

```
git commit -m "Выполнил задание 1.2"
```

Коммит фиксирует текущее состояние файлов и сохраняет историю изменений.

> Сообщение коммита должно кратко и понятно описывать внесённые изменения.

---

## Шаг 7. Отправка изменений

После создания коммита необходимо отправить изменения
в удалённый репозиторий на GitHub.

Выполните команду:

```
git push
```

Команда отправляет ваши коммиты из локального репозитория
в удалённый репозиторий (ваш fork на GitHub).

> После выполнения `git push` изменения станут доступны на GitHub
> и могут быть использованы для создания Pull Request.

## Полезные команды (шпаргалка)

Ниже приведены основные команды Git, которые используются при работе
с репозиторием в процессе выполнения заданий.

---

### Основные команды Git

| Команда | Назначение |
|--------|-----------|
| `git status` | Показать текущее состояние репозитория: изменённые файлы, новые файлы, подготовленные к коммиту изменения |
| `git pull` | Получить и применить изменения из удалённого репозитория |
| `git add .` | Добавить все изменения в индекс (подготовить к коммиту) |
| `git commit -m "сообщение"` | Зафиксировать изменения в локальном репозитории |
| `git push` | Отправить локальные коммиты в удалённый репозиторий |
| `git log` | Просмотреть историю коммитов |
| `git branch` | Просмотреть список веток |
| `git checkout <ветка>` | Переключиться на другую ветку |
| `git checkout -b <ветка>` | Создать новую ветку и перейти в неё |

---

### Когда использовать команды (практический порядок)

| Ситуация | Что делать |
|--------|-----------|
| Начинаете работу над заданием | `git pull` — обновить проект |
| Проверить, что изменилось | `git status` |
| Добавили или изменили файлы | `git add .` |
| Сохранить прогресс | `git commit -m "описание изменений"` |
| Отправить работу на GitHub | `git push` |
| Хотите проверить историю изменений | `git log` |
| Нужно начать новое задание в отдельной ветке | `git checkout -b <название-ветки>` |

---

## Важно

- Выполняйте `git pull` **перед началом работы**, чтобы не возникло конфликтов.
- Используйте `git status`, чтобы понимать, какие файлы изменены.
- После завершения задания выполните:
  - `git add .`
  - `git commit -m "описание"`
  - `git push`

Это базовый и безопасный workflow для работы с Git.
