# Git и GitHub

Git и GitHub нужны для хранения кода, отслеживания изменений и отправки выполненных заданий в удалённый репозиторий.

---

## Материалы модуля

| № | Материал | Краткое описание |
|---|---|---|
| 1 | [Установка Git на Windows](<./Установка Git на Windows.md>) | Пошаговая установка Git for Windows: загрузка установщика, выбор основных параметров, установка Git Bash и проверка команды `git --version`. |
| 2 | [Регистрация аккаунта GitHub](<./Регистрация аккаунта GitHub.md>) | Создание аккаунта GitHub: ввод почты, пароля и имени пользователя, прохождение проверки, подтверждение электронной почты и первый вход в аккаунт. |
| 3 | [Как сделать fork репозитория](<./Как сделать fork репозитория.md>) | Создание собственной копии учебного репозитория в GitHub-аккаунте. Объясняется, зачем нужен fork и как проверить, что открыта именно своя копия проекта. |
| 4 | [Основные команды для работы с Git через консоль](<./Основные команды для работы с Git через консоль(Git Bash).md>) | Базовый рабочий процесс через Git Bash: `git clone`, `git status`, `git pull`, `git add`, `git commit`, `git push`. Также есть краткая шпаргалка по командам. |
| 5 | [Создание Pull Request](<./Создание Pull Request.md>) | Создание Pull Request для отправки выполненного задания на проверку: открытие fork, проверка направления изменений, заполнение названия и описания PR. |
| 6 | [Как выполнять и сдавать домашние задания](<./Как выполнять и сдавать домашние задания.md>) | Полный алгоритм сдачи домашней работы: обновление репозитория, создание папки для задания, выполнение работы, commit, push, Pull Request и исправление замечаний. |
