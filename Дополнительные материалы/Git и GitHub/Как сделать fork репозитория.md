# Как сделать fork репозитория

---

**Fork репозитория** — это создание полной копии существующего репозитория в вашем аккаунте GitHub.

Fork создаёт независимую версию проекта в вашем аккаунте, сохраняя связь с оригинальным репозиторием.

Fork используется для:

- самостоятельной работы с кодом без риска повредить основной репозиторий;
- выполнения учебных заданий и практических задач;
- экспериментов с кодом, добавления нового функционала или исправления ошибок;
- подготовки изменений для последующей отправки в оригинальный репозиторий через Pull Request;
- участия в командной разработке и Open Source-проектах.

## Шаг 3. Настройка fork

После нажатия кнопки **Fork** откроется страница создания копии репозитория.

GitHub предложит настроить параметры fork перед созданием.

### Основные поля

🔹 **Owner (владелец)**  
Выберите аккаунт, в котором будет создан fork.

> Обычно это ваш личный аккаунт

---

🔹 **Repository name (имя репозитория)**  
По умолчанию имя совпадает с оригинальным репозиторием.

> Рекомендуется оставить без изменений, чтобы не было путаницы

---

🔹 **Description (описание)**  
Автоматически копируется из оригинального репозитория.

> Можно:
- оставить как есть;
- изменить (не обязательно)

---

### Дополнительная настройка

🔹 **Copy the main branch only**

Если включено:
- копируется только ветка `main`

Если выключено:
- копируются все ветки

> ✅ Рекомендуется оставить включённым (проще для начала)

### Создание fork

Нажмите кнопку:

> **Create fork**

## Шаг 4. Результат

После создания fork вы будете перенаправлены
в копию репозитория в вашем аккаунте.

![img_3.png](../%D0%A1%D0%BA%D1%80%D0%B8%D0%BD%D1%88%D0%BE%D1%82%D1%8B/git%20%26%20github/%D0%9A%D0%B0%D0%BA%20%D1%81%D0%B4%D0%B5%D0%BB%D0%B0%D1%82%D1%8C%20fork%20%D1%80%D0%B5%D0%BF%D0%BE%D0%B7%D0%B8%D1%82%D0%BE%D1%80%D0%B8%D1%8F/img_3.png)

В верхней части страницы вы увидите:

- название репозитория;
- пометку `forked from <оригинальный репозиторий>` — это означает, что данный репозиторий является копией;
- ваш аккаунт как владельца.

### Что важно проверить

Убедитесь, что вы находитесь **в своей копии**, а не в оригинальном репозитории.  
Это можно проверить по:

- имени владельца (слева вверху);
- наличию пометки `forked from`.

### Что теперь доступно

После создания fork вы можете:

- работать с кодом в своей копии;
- выполнять задания;
- отправлять изменения через Pull Request.
