# Как выполнять и сдавать домашние задания

Домашние задания в курсе сдаются через **fork репозитория** и **Pull Request**.

Fork создаётся **один раз** в начале работы с курсом.  
После этого все домашние задания выполняются в вашей копии репозитория и отправляются на проверку через Pull Request.

---

## Общая схема

```
Один раз сделать fork репозитория
↓
Один раз склонировать свой fork на компьютер
↓
Перед выполнением задания обновить репозиторий
↓
Создать папку для своей домашней работы
↓
Выполнить задание
↓
Проверить изменения
↓
Добавить изменения и создать commit
↓
Отправить изменения на GitHub
↓
Создать Pull Request
↓
Дождаться проверки
↓
Исправить замечания, если они есть
```

---

## 1. Один раз сделать fork репозитория

Перед началом работы с курсом нужно создать свою копию учебного репозитория.

Для этого используется **fork**.

Инструкция:

[Как сделать fork репозитория](<./Как сделать fork репозитория.md>)

Fork нужно сделать **один раз**.  
После этого все задания выполняются уже в вашей копии репозитория.

---

## 2. Один раз склонировать свой fork на компьютер

После создания fork нужно скачать его на компьютер.

Откройте свой fork на GitHub.

Нажмите:

```
Code → HTTPS → скопировать ссылку
```

Затем откройте **Git Bash** в папке, где хотите хранить проект, и выполните команду:

```
git clone <ссылка-на-ваш-fork>
```

Пример:

```
git clone https://github.com/username/programming-foundations.git
```

После клонирования перейдите в папку проекта:

```
cd programming-foundations
```

Клонирование тоже выполняется **один раз**.  
Дальше вы просто открываете уже скачанную папку проекта на компьютере.

---

## 3. Найти практическое задание

Практические задания находятся в соответствующем разделе курса.

Например, для раздела **«Основы программирования на C (ОП)»**:

```text
Основы программирования на C (ОП)
└── Практические задания
    └── 1. Вступление
        └── Практическое задание.md
```

Перед выполнением задания внимательно прочитайте:

- тему задания;
- цель задания;
- что нужно сделать;
- что нужно сдать;
- критерии выполнения.

---

## 4. Перед выполнением задания обновить репозиторий

Перед началом работы над новым заданием нужно получить актуальные изменения из GitHub.

Для этого в папке проекта выполните:

```
git pull
```

Это нужно, чтобы работать с последней версией материалов курса.

> Делайте `git pull` перед каждым новым заданием.

---

## 5. Создать папку для своей домашней работы

Все домашние работы нужно добавлять в папку:

```text
Домашние работы
```

Внутри создайте папку со своим именем и фамилией.

Общий шаблон:

```text
Домашние работы
└── Фамилия Имя
    └── Номер и название модуля
        └── Название задания
```

Пример для первого задания:

```text
Домашние работы
└── Иванов Иван
    └── 1. Вступление
        └── Практическое задание
            ├── first_program.c
            └── result.png
```

Где:

| Файл | Что это |
|---|---|
| `first_program.c` | исходный код программы |
| `result.png` | скриншот результата запуска программы |

---

## 6. Выполнить задание

Выполните задание по инструкции из файла практической работы.

Важно:

- не изменяйте файлы с теорией;
- не удаляйте чужие файлы;
- не добавляйте временные файлы среды разработки, если они не требуются;
- кладите решение только в свою папку внутри `Домашние работы`.

---

## 7. Проверить изменения

После выполнения задания проверьте состояние репозитория:

```
git status
```

Команда покажет:

- какие файлы были добавлены;
- какие файлы были изменены;
- какие файлы ещё не добавлены в commit.

Перед отправкой обязательно проверьте, что в список изменений попали только файлы вашей домашней работы.

---

## 8. Добавить изменения в индекс

Если всё корректно, добавьте изменения:

```
git add .
```

После этого можно ещё раз проверить состояние:

```
git status
```

---

## 9. Создать commit

Создайте commit с понятным сообщением.

Пример:

```
git commit -m "Выполнил домашнее задание по модулю 1"
```

Можно писать конкретнее:

```
git commit -m "Добавил первую программу на C"
```

Хорошее сообщение commit должно кратко объяснять, что было сделано.

---

## 10. Отправить изменения на GitHub

После создания commit отправьте изменения в свой fork:

```
git push
```

Если Git попросит авторизоваться, войдите в свой GitHub-аккаунт.

После выполнения `git push` изменения появятся в вашем fork на GitHub.

---

## 11. Создать Pull Request

После отправки изменений на GitHub нужно создать **Pull Request**.

Инструкция:

[Создание Pull Request](<./Создание Pull Request.md>)

В Pull Request нужно указать понятное название.

Примеры:

```text
Домашнее задание по модулю 1
```

```text
Практическое задание 1 — первая программа на C
```

```text
Выполнил задание 1.2 — первая программа
```

В описании Pull Request кратко напишите, что было сделано.

Пример:

```text
Что сделано:
- создана папка для домашней работы;
- добавлен исходный файл first_program.c;
- добавлен скриншот результата запуска программы.

Проверка:
- программа запускается без ошибок;
- в консоли отображается сообщение.
```

---

## 12. Дождаться проверки

После создания Pull Request работа отправляется на проверку.

Проверяющий может:

- принять работу;
- оставить комментарии;
- попросить исправить ошибки;
- закрыть Pull Request, если работа оформлена неправильно.

---

## 13. Исправить замечания

Если проверяющий оставил замечания, **не создавайте новый Pull Request**.

Исправления нужно внести в те же файлы и отправить новым commit.

После исправлений выполните:

```
git add .
git commit -m "Исправил замечания по домашнему заданию"
git push
```

После команды `git push` Pull Request обновится автоматически.

---

## Частые ошибки

| Ошибка | Что не так | Как исправить |
|---|---|---|
| Fork создаётся заново перед каждым заданием | Fork нужен только один раз в начале курса | Используйте уже созданный fork |
| Работа добавлена в папку с теорией | Учебные материалы засоряются файлами учеников | Кладите решение в `Домашние работы/<Фамилия Имя>/...` |
| В Pull Request попали лишние файлы | Проверяющему сложнее смотреть работу | Перед commit проверяйте `git status` |
| Не добавлен скриншот результата | Нельзя быстро проверить запуск программы | Добавьте скриншот в папку задания |
| Создан новый Pull Request после замечаний | История проверки распадается | Исправляйте файлы и делайте `git push` |
| Название Pull Request слишком общее | Непонятно, какое задание отправлено | Указывайте номер модуля или урока |
| Перед заданием не выполнен `git pull` | Локальная копия может быть устаревшей | Перед каждым заданием обновляйте репозиторий |

## Краткая шпаргалка команд

### Первый раз

```
git clone <ссылка-на-ваш-fork>
cd programming-foundations
```

### Перед каждым заданием

```
git pull
git status
```

### После выполнения задания

```
git add .
git commit -m "Выполнил домашнее задание по модулю 1"
git push
```

После этого на GitHub создаётся Pull Request.

---

## Мини-чеклист перед отправкой

Перед созданием Pull Request проверьте:

- [ ] задание выполнено полностью;
- [ ] файлы лежат в папке `Домашние работы`;
- [ ] создана папка с вашей фамилией и именем;
- [ ] исходный код добавлен;
- [ ] скриншот результата добавлен, если он требуется;
- [ ] программа запускается без ошибок;
- [ ] `git status` не показывает лишних файлов;
- [ ] commit создан;
- [ ] изменения отправлены через `git push`;
- [ ] Pull Request создан.

---

## Итог

Домашняя работа считается отправленной, когда:

1. решение лежит в правильной папке;
2. изменения отправлены в ваш fork;
3. создан Pull Request в основной репозиторий;
4. Pull Request доступен для проверки.

Главная схема:

```text
обновил репозиторий → сделал задание → commit → push → Pull Request → проверка
```
