# Как работает программа на C

## Общая схема

Программа на C проходит несколько основных этапов:

```
Исходный код → Компиляция → Исполняемый файл → Запуск программы
```

Если совсем коротко:

1. **Исходный код** — мы пишем программу в файле `.c`.
2. **Компиляция** — компилятор проверяет код и переводит его в понятный компьютеру вид.
3. **Исполняемый файл** — после успешной компиляции появляется готовая программа.
4. **Запуск программы** — пользователь запускает программу и получает результат.

---

## 1. Исходный код

Сначала программист пишет обычный текстовый файл с кодом на языке C.

Обычно такой файл имеет расширение:

```
.c
```

Например:

```
main.c
```

Пример простой программы:

```
#include <stdio.h>

int main() {
    printf("Hello, World!");
    return 0;
}
```

На этом этапе важно понимать простую вещь:

> исходный код — это текст, который понятен человеку, но компьютер не может выполнить его напрямую.

Компьютеру нужно сначала перевести этот код в более низкоуровневый вид.

---

## 2. Компиляция

Чтобы программа заработала, исходный код нужно скомпилировать.

**Компиляция** — это процесс преобразования кода на C в исполняемую программу.

За это отвечает специальная программа — **компилятор**.

Компилятор выполняет две основные задачи:

1. Проверяет код на ошибки.
2. Переводит код в форму, которую может выполнить компьютер.

Пример команды компиляции:

```
gcc main.c -o main
```

Здесь:

| Часть команды | Значение |
|---|---|
| `gcc` | название компилятора |
| `main.c` | файл с исходным кодом |
| `-o main` | имя будущей программы |

В Visual Studio этот процесс обычно запускается кнопкой **Start** или **Build**.  
То есть Visual Studio сама вызывает компилятор, и студенту не нужно каждый раз писать команду вручную.

---

## 3. Исполняемый файл

Если в коде нет ошибок, после компиляции появляется исполняемый файл.

На Windows это обычно файл с расширением:

```
.exe
```

Например:

```text
main.exe
```

Это уже не просто текстовый файл.  
Это готовая программа, которую операционная система может запустить.

Важно:

> файл `.c` — это код программы, а файл `.exe` — это уже собранная программа.

---

## 4. Запуск программы

После компиляции программу можно запустить.

При запуске операционная система загружает исполняемый файл и начинает выполнять инструкции программы.

Например, если в программе написано:

```
printf("Hello, World!");
```

то на экране появится:

```text
Hello, World!
```

То есть итоговый путь выглядит так:

```text
написали код → скомпилировали → получили программу → запустили
```

---

## Что происходит, если в коде ошибка

Если в программе есть ошибка, компилятор сообщит об этом.

Например:

```
printf("Hello, World!")
```

Здесь пропущена точка с запятой `;`.

Компилятор может вывести сообщение об ошибке примерно такого вида:

```text
error: expected ';'
```

В этом случае исполняемый файл не будет создан, пока ошибка не исправлена.

Это нормально. Ошибки компиляции — обычная часть обучения программированию.

---

## Главное, что нужно запомнить

- C — это **компилируемый язык программирования**.
- Компьютер не выполняет файл `.c` напрямую.
- Сначала код нужно скомпилировать.
- После компиляции получается исполняемый файл.
- Если в коде есть ошибка, компилятор покажет сообщение и программу нужно исправить.
